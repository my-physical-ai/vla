1 모델 구조 — "AI의 뇌 크기 설계"

뇌가 클수록 복잡한 동작을 배울 수 있지만, GPU 메모리와 학습 시간이 더 필요합니다.

🧠
비유: 학생의 뇌 크기

dim_model은 뇌의 크기, dim_feedforward는 뇌 안의 고속도로 폭, encoder/decoder 층수는 생각의 깊이, dropout은 일부러 눈을 가리고 공부하기(과적합 방지)예요.

🧠 dim_model — Transformer 내부 차원

설정값 뇌 크기 표현력 GPU 메모리 학습 속도 적합한 상황
64 🧠 아주 작음 ⭐ ~1GB 🚀 매우 빠름 디버깅/테스트용
128 🧠 작음 ⭐⭐ ~2GB 🚀 빠름 단순 태스크 (1가지 동작)
256 🧠🧠 보통 ⭐⭐⭐ ~3GB ⚡ 보통 일반적인 pick & place
384 🧠🧠🧠 큼 ⭐⭐⭐⭐ ~5GB 🐢 느림 복잡한 동작 (추천)
512 🧠🧠🧠🧠 매우 큼 ⭐⭐⭐⭐⭐ ~8GB 🐢 매우 느림 대규모 데이터 + 고성능 GPU

🛣️ dim_feedforward — FFN 차원 (뇌 안의 고속도로 폭)

설정값 dim_model 대비 처리 능력 적합한 상황
256×1 (동일)⭐⭐아주 가벼운 모델
512×2⭐⭐⭐경량 모델
1024×2.7 (dim=384)⭐⭐⭐⭐균형 잡힌 선택 (추천)
1536×4⭐⭐⭐⭐⭐충분한 GPU 여유 시
2048×4 (dim=512)⭐⭐⭐⭐⭐대형 모델용
💡
공식: dim_feedforward = dim_model × 2~4배가 일반적. 384 × 2.7 ≈ 1024가 우리 설정이에요.

🏗️ 인코더/디코더 층수 — 생각의 깊이

파라미터 1층 3층 5층 8층
n_encoder_layers
"이해" 깊이		 피상적 이해 충분한 이해 (추천) 깊은 이해 과도 (과적합 위험)
n_decoder_layers
"동작 생성" 깊이		 충분 (추천) 복잡한 동작용 과도 불필요
💡
왜 인코더 > 디코더?

수학 문제를 "이해하는 것"(인코더)이 어렵지, 이해하고 나면 "답 쓰는 것"(디코더)은 쉬워요! 그래서 인코더=3, 디코더=1이 효율적이에요.

🙈 dropout — 과적합 방지율

설정값 뇌 사용률 효과 데이터 적을 때 데이터 많을 때
0.0100%과적합 방지 없음❌ 외움✅ OK
0.0595%약한 방지⚠️ 부족✅ 좋음
0.190%균형 (추천)✅ 적당✅ 좋음
0.280%강한 방지✅ 좋음⚠️ 과도
0.370%매우 강함✅ 데이터<10개❌ 성능 저하
0.550%극단적⚠️ 너무 느림❌ 성능 저하

2 동작 생성 — "한 번에 얼마나 계획할까?"

ACT의 핵심! chunk_size로 멀리 계획하고, n_action_steps로 가까이만 실행합니다.

🗺️
비유: 내비게이션 운전

chunk_size = 내비가 보여주는 전체 경로 (멀리까지 계획)
n_action_steps = 실제로 핸들을 조작하는 구간 (가까이만 실행)
멀리 보면서 가까이 조절하면 부드러우면서도 정확한 운전이 돼요!

📋 chunk_size — 한 번에 계획하는 미래 스텝 수

설정값 시간 환산
(30fps)		 동작 부드러움 반응 속도 적합한 태스크
100.3초⭐⭐ 끊김🚀 즉각 반응빠른 반사 동작
200.7초⭐⭐⭐⚡ 빠름단순 pick & place
501.7초⭐⭐⭐⭐ 부드러움🐢 보통복잡한 조작 (추천)
1003.3초⭐⭐⭐⭐⭐ 매우 부드러움🐢 느림긴 연속 동작

🏃 n_action_steps — 실제 실행하는 스텝 수

설정값 chunk 대비
(chunk=50)		 재계획 빈도 정확도 계산 부담
510%만 실행🔄 매우 자주🎯 매우 높음💻 높음 (추론 많음)
1020%🔄 자주🎯 높음💻 보통
2040%🔄 적당🎯 적당 (추천)💻 낮음
50100% 전부 실행❌ 재계획 없음⚠️ 낮음💻 최저
chunk_size=50, n_action_steps=20 시각화
50개를 계획하되 → 앞 20개만 실행 → 다시 새로 계획 (반복)
1차 계획:
실행 (20개)
버림 (30개)
2차 계획 (새 위치에서):
실행 (20개)
버림 (30개)
3차 계획 (새 위치에서):
실행 (20개)
버림 (30개)
→ 이것이 ACT가 부드러운 동작을 만드는 비결!

🎛️ kl_weight — 다양성 vs 일관성 조절

설정값 성격 동작 결과 비유
0.01매우 자유롭게매번 다른 경로 → 불안정할 수 있음아무데나 글씨 쓰기
0.05자유롭게다양한 접근 경로 허용줄 근처에 자유롭게
0.1균형 (추천)적당한 다양성 + 안정성줄에 맞추되 유연하게
0.5일관되게거의 같은 경로 반복줄에 딱 맞춰 쓰기
1.0매우 일관항상 동일한 동작 → 유연성 부족인쇄처럼 정확하게

3 비전 백본 — "AI의 눈 설정"

카메라 이미지를 이해하는 신경망. 어떤 안경(백본)을 쓰고, 동결할지 말지가 핵심입니다.

👓
비유: 전학생의 안경

ImageNet에서 전학 온 학생이 기존 안경(사전학습)을 쓰고 있어요. 동결 = 안경 그대로 유지, 미세조정 = 새 환경에 맞게 안경 도수 조절!

🔬 vision_backbone — 이미지 인식 네트워크

백본 파라미터 수 인식 성능 속도 GPU 메모리 적합한 상황
resnet18 11M ⭐⭐⭐ 🚀 빠름 ~2GB 교육/실험 (추천)
resnet34 21M ⭐⭐⭐⭐ ⚡ 보통 ~3GB 중간 복잡도
resnet50 25M ⭐⭐⭐⭐⭐ 🐢 느림 ~5GB 세밀한 인식 필요 시

🔒 optimizer_lr_backbone — 눈 동결 여부

설정값 상태 눈(백본) 변화 색상 구분 적합한 상황
0.0 🔒 완전 동결 전혀 안 바뀜 ❌ 못 배움 형태 구분만 필요할 때
1e-6 🔓 극미세 조정 아주 살짝 ⚠️ 약간 안전한 미세 조정
1e-5 🔓 미세 조정 (추천) 적절히 적응 ✅ 가능 색상 구분 태스크
1e-4 🔓 본격 학습 크게 변화 ✅ 강함 데이터 많을 때만
🚨
주의! "분홍색 블록을 노란 상자에" 같은 색상 구분 태스크에서 backbone_lr=0.0(동결)이면 색상 차이를 학습하지 못합니다. 이 경우 반드시 freeze_vision_encoder=false 또는 backbone_lr=1e-5 이상으로 설정하세요! (관련: GitHub Issue #1774)

4 옵티마이저 — "공부 방법 설정"

학습률, 가중치 감쇠, 그래디언트 클리핑 — AI가 어떤 보폭으로, 얼마나 안정적으로 배울지 결정합니다.

👣
비유: 산에서 가장 낮은 곳 찾기

AI 학습은 안개 낀 산에서 가장 낮은 골짜기(최적값)를 찾는 것과 같아요. lr(학습률) = 한 걸음의 보폭, weight_decay = 무거운 짐 내려놓기, grad_clip = 절벽에서 뛰어내리기 방지!

👣 optimizer.lr — 학습률 (보폭 크기)

설정값 보폭 학습 속도 안정성 용도
1e-2🏃 매우 큰 보폭🚀 매우 빠름❌ 불안정 (발산 위험)거의 사용 안 함
1e-3🚶 큰 보폭⚡ 빠름⚠️ 보통처음부터 학습 시
1e-4🐢 작은 보폭🐢 보통✅ 안정적 (추천)사전학습 모델 활용 시
1e-5🐌 아주 작은 보폭🐌 느림✅ 매우 안정미세 조정 (Fine-tuning)
1e-6🔬 극소 보폭🔬 매우 느림✅ 최고 안정최종 마무리 단계

🧹 weight_decay — 가중치 감쇠 (불필요한 암기 지우기)

설정값 정리 강도 과적합 방지 성능 영향
0지우개 없음❌ 방지 안 됨과적합 시 성능 저하
1e-4가끔 지우기 (추천)✅ 적당일반적으로 최적
1e-3자주 지우기✅ 강함데이터 적을 때 유용
1e-2너무 자주⚠️ 과도중요한 것도 지워질 수 있음

🚦 grad_clip_norm — 그래디언트 폭발 방지

설정값 제한 강도 안정성 설명
없음제한 없음❌ 폭발 위험그래디언트가 무한히 커질 수 있음
0.5엄격 (추천)✅ 매우 안정수정량 최대 0.5로 제한
1.0보통✅ 안정일반적인 Transformer 기본값
5.0느슨함⚠️ 보통거의 제한 안 함

5 학습 스케줄 — "공부 계획표"

총 스텝 수, 워밍업, 코사인 감쇠 — 학습률을 시간에 따라 어떻게 변화시킬지 설정합니다.

📅
비유: 마라톤 페이스 전략

Warmup = 출발 후 몸 풀며 천천히 속도 올리기, Peak = 최고 속도로 달리기, Decay = 결승선 앞에서 서서히 속도 줄이며 마무리. 처음부터 전력질주하면 금방 지치고, 끝까지 전력이면 마무리가 거칠어요!

파라미터들은 독립적이지 않고, 서로 영향을 미쳐요.

① dim_model ↑ → 메모리 ↑ → batch_size ↓ 필요
뇌가 커지면 GPU 메모리를 많이 써서, 한 번에 처리하는 데이터를 줄여야 해요.

② chunk_size ↑ → 부드러움 ↑ → 학습 난이도 ↑
멀리 계획할수록 부드럽지만, AI가 예측해야 할 양이 많아져 학습이 어려워져요.

③ dropout ↑ + 데이터 ↓ → 더 나은 일반화
데이터가 적으면 dropout을 높여서 과적합을 방지하는 게 효과적이에요.

④ lr ↑ + grad_clip ↓ → 불안정 위험
큰 보폭 + 제한 없음은 학습이 폭발할 수 있어요. 반드시 함께 조절!

⑤ backbone 동결 + 색상 태스크 → 실패 위험!
ImageNet 특징은 형태 중심이라, 색상만 다른 물체 구분에 실패합니다.
데이터(에피소드) 수 추천 dim_model 추천 dropout 추천 steps
5~10개1280.2~0.310K~20K
20~30개256~3840.1~0.250K~100K
50~100개384~5120.05~0.1100K~200K
200개+5120.0~0.05200K+

데이터가 적으면 작은 모델 + 높은 dropout, 데이터가 많으면 큰 모델 + 낮은 dropout이 원칙이에요.

📌 핵심 정리

1
dim_model / dim_feedforward는 뇌의 크기와 고속도로 폭. 384/1024 조합이 SO-101 실험의 균형점.
2
인코더 3 > 디코더 1 — "이해"가 "실행"보다 더 깊은 처리가 필요.
3
chunk_size=50, n_action_steps=20 — 멀리 계획하고(50) 가까이만 실행(20)해서 부드러움 + 정확함 확보.
4
색상 구분 태스크는 backbone 동결 해제 필수! backbone_lr=0.0이면 색상을 학습하지 못함. (Issue #1774)
5
cosine_decay_with_warmup 스케줄러로 "준비 → 전력 → 마무리" 패턴 적용. warmup 5% 권장.
6
파라미터는 서로 연동됨. 데이터 수 → 모델 크기 → dropout → steps를 함께 조절해야 최적의 결과를 얻을 수 있음.

🔍 문제 요약
문제 분류
모델 캐시 이동/복사 시 에러
추정 원인
심볼릭 링크 깨짐 + 메타데이터 불일치
영향 범위
SmolVLA 모델 오프라인 배포
🔬 에러 원인 분석 (3가지 원인 교차 확인)

원인 1: 심볼릭 링크(Symlink) 깨짐 — 7개 사이트에서 교차 확인

HuggingFace 캐시(~/.cache/huggingface/hub/)의 내부 구조는 blobs/에 실제 파일이 있고, snapshots/해시/ 안의 파일들은 심볼릭 링크로 blob을 가리킵니다. USB(FAT32/exFAT)는 심볼릭 링크를 지원하지 않아서 cp 명령으로 복사하면 링크가 깨지거나 0바이트 파일이 됩니다.

원인 2: 메타데이터/리비전 불일치 — 5개 사이트에서 교차 확인

캐시 폴더의 refs/main 파일이 커밋 해시를 가리키는데, 이 해시와 snapshots/ 폴더명이 일치해야 합니다. 복사 과정에서 이 매핑이 깨지면 LocalEntryNotFoundError: Cannot find an appropriate cached snapshot folder 에러가 발생합니다.

원인 3: Hub 버전 체크 시도 — 4개 사이트에서 교차 확인

캐시에서 로드하더라도 huggingface_hub는 기본적으로 HTTP 요청을 보내 새 버전이 있는지 확인합니다. 오프라인 환경(교육생 PC)에서 이 요청이 실패하면 에러가 발생합니다. HF_HUB_OFFLINE=1 환경변수가 필요합니다.

💡 해결책 (우선순위 순)
1순위 — 가장 확실한 방법 (교육 현장 추천)

snapshot_download --local-dir 로 "클린 폴더" 생성 후 USB 복사

캐시 폴더를 직접 복사하지 말고, 심볼릭 링크 없는 독립 폴더를 만들어서 복사합니다. HuggingFace 공식 권장 방법입니다.

선생님 PC에서 실행 (인터넷 있는 환경, 1회만):

# Python 스크립트로 클린 폴더 생성
python3 -c "
from huggingface_hub import snapshot_download

# SmolVLA fine-tuned 모델
snapshot_download(
    repo_id='ch1234bigman/pink_task_smolvla_model',
    local_dir='/home/zetabank/usb_models/pink_task_smolvla_model'
)
"

USB에 복사:

# USB 마운트 확인
lsblk

# 클린 폴더를 USB로 복사 (심볼릭 링크 없으므로 안전)
cp -r /home/zetabank/usb_models/pink_task_smolvla_model /media/zetabank/USB이름/

교육생 PC에서 실행:

# USB에서 원하는 위치로 복사
cp -r /media/학생이름/USB이름/pink_task_smolvla_model ~/my_models/

# 로컬 경로로 직접 실행 (HF 로그인 불필요!)
lerobot-rollout \
  --strategy.type=dagger \
  --strategy.num_episodes=20 \
  --policy.path=~/my_models/pink_task_smolvla_model \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \
  --robot.cameras="{ front: {type: opencv, index_or_path: 6, width: 640, height: 480, fps: 30} }" \
  --teleop.type=so101_leader \
  --teleop.port=/dev/ttyACM1 \
  --dataset.repo_id=data/dagger_corrections \
  --dataset.single_task="pink block into the box of yellow" \
  --dataset.push_to_hub=false \
  --display_data=true

핵심: --policy.path에 HuggingFace repo_id 대신 로컬 폴더 경로를 넣으면 됩니다.

2순위 — CLI 원라이너 방식

hf download 명령어로 클린 폴더 생성

Python 스크립트 대신 터미널 한 줄로 동일한 효과를 얻을 수 있습니다.

# 선생님 PC에서 (인터넷 환경)
hf download ch1234bigman/pink_task_smolvla_model \
  --local-dir /home/zetabank/usb_models/pink_task_smolvla_model

결과 폴더에는 config.json, model.safetensors, config.yaml 등이 직접 들어있어서 USB 복사가 안전합니다.

3순위 — 캐시 폴더를 반드시 복사해야 할 경우

cp -rL (심볼릭 링크 실체화) + HF_HUB_OFFLINE=1

캐시 구조를 유지하면서 복사해야 한다면 -L 옵션으로 심볼릭 링크를 실제 파일로 변환합니다.

# -L 옵션이 핵심! 심볼릭 링크를 실제 파일로 복사
cp -rL ~/.cache/huggingface/hub/models--ch1234bigman--pink_task_smolvla_model /media/zetabank/USB이름/

# 교육생 PC에서 캐시 폴더에 넣기
mkdir -p ~/.cache/huggingface/hub/
cp -r /media/학생이름/USB이름/models--ch1234bigman--pink_task_smolvla_model ~/.cache/huggingface/hub/

# 오프라인 모드 활성화 (Hub 접속 시도 차단)
echo 'export HF_HUB_OFFLINE=1' >> ~/.bashrc
source ~/.bashrc

⚠️ 주의: 의존 모델(SmolVLA base 등)도 함께 복사해야 합니다. ls ~/.cache/huggingface/hub/ | grep smolvla로 확인하세요.

🎓 교육 현장용 배포 스크립트 (선생님용)

이 스크립트를 선생님 PC에서 한 번 실행하면, USB에 교육생 배포용 클린 모델 폴더가 만들어집니다.

#!/bin/bash
# prepare_student_models.sh — 교육생 배포용 모델 준비
# 선생님 PC에서 실행 (인터넷 + HuggingFace 로그인 필요)

USB_PATH="/media/zetabank/USB이름"  # USB 경로 수정
TEMP_DIR="/home/zetabank/student_models"

echo "📦 [1/3] 모델 다운로드 (클린 폴더로)..."
mkdir -p ${TEMP_DIR}

python3 -c "
from huggingface_hub import snapshot_download
snapshot_download(
    repo_id='ch1234bigman/pink_task_smolvla_model',
    local_dir='${TEMP_DIR}/pink_task_smolvla_model'
)
print('✅ 다운로드 완료')
"

echo "📋 [2/3] USB로 복사..."
cp -r ${TEMP_DIR}/pink_task_smolvla_model ${USB_PATH}/
echo "✅ USB 복사 완료"

echo "📝 [3/3] 교육생 설치 스크립트 생성..."
cat > ${USB_PATH}/install_model.sh << 'SCRIPT'
#!/bin/bash
# 교육생 PC에서 실행
MODEL_DIR="$HOME/lerobot_models"
mkdir -p ${MODEL_DIR}
cp -r "$(dirname "$0")/pink_task_smolvla_model" ${MODEL_DIR}/
echo "✅ 모델 설치 완료!"
echo "📁 경로: ${MODEL_DIR}/pink_task_smolvla_model"
echo ""
echo "🤖 실행 예시:"
echo "lerobot-rollout --policy.path=${MODEL_DIR}/pink_task_smolvla_model ..."
SCRIPT
chmod +x ${USB_PATH}/install_model.sh

echo ""
echo "🎉 완료! USB를 교육생에게 배포하세요."
echo "교육생은 USB에서 ./install_model.sh 실행 후 사용 가능합니다."

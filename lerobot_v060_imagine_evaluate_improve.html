0 핵심 개념: 로봇 학습 순환 고리

v0.6.0은 로봇이 스스로 배우는 "순환 고리"를 완성했습니다. 마치 사람이 시험 → 채점 → 오답 노트 → 다시 시험하는 것처럼요!

📝
시험 공부에 비유하면!

여러분이 수학 시험 공부를 한다고 상상해 보세요.
① 상상(Imagine) — 문제를 풀기 전에 "이렇게 풀면 답이 나올까?" 머릿속으로 미리 생각해봐요
② 평가(Evaluate) — 답을 쓴 뒤 채점해서 맞았는지 확인해요
③ 개선(Improve) — 틀린 문제는 오답노트를 쓰고, 다시 풀어봐요

💡 로봇도 똑같아요! — 행동 전에 미래를 예측하고(World Model), 성공 여부를 자동 채점하고(Reward Model), 실패한 부분은 사람이 교정해주면 다시 학습합니다(DAgger).

1 🌍 상상하라 — 월드 모델 (World Model)

"행동하기 전에 미래를 먼저 상상하는" 로봇 정책 3종이 추가되었습니다

🧠
쉽게 이해하기!

바둑이나 장기를 둘 때, 고수는 돌을 놓기 전에 "이 수를 두면 다음에 상대가 이렇게 나올 거고…" 하고 여러 수 앞을 머릿속으로 미리 시뮬레이션합니다. 월드 모델도 같은 원리예요! 로봇이 "컵을 잡으면 다음에 이렇게 될 거야"라고 미래를 상상한 뒤, 가장 좋은 행동을 선택합니다.

💡 핵심 질문 — "월드 모델이 정말 로봇 정책에 도움이 될까?" v0.6.0은 이 질문에 답하기 위해 3가지 다른 접근법을 제공합니다.

🎯
VLA-JEPA
훈련할 때는 미래를 예측하는 월드 모델이 함께 공부하지만, 실제 로봇에서 사용할 때는 월드 모델을 제거합니다. 마치 시험 볼 때는 참고서 없이 실력만으로 푸는 것처럼!
추론 비용 제로
🎬
LingBot-VA
미래 영상과 행동을 한 묶음씩(Chunk) 예측합니다. 상상한 영상과 실제 영상을 비교할 수도 있어요! "로봇이 뭘 상상했는지" 직접 볼 수 있는 유일한 모델.
영상 예측 저장 가능
💭
FastWAM
~50억 파라미터의 영상 생성 전문가가 훈련 중 "꿈을 꿉니다." 실제 사용할 때는 꿈꾸기를 건너뛰고 바로 행동합니다. 훈련의 상상력 + 실행의 속도를 동시에!
~5B 파라미터
💡
공통점: 세 모델 모두 훈련할 때는 미래를 상상하지만, 실제 로봇에서는 상상을 건너뛰어 추가 비용 없이 빠르게 행동합니다. 상상은 "더 좋은 뇌를 만드는 훈련법"이지, "느려지는 원인"이 아닙니다!

3 🏅 평가하라 — 보상 모델 (Reward Model)

"로봇이 성공했는지" 자동으로 채점해주는 AI가 추가되었습니다

✅
자동 채점기!

시험지를 채점할 때, 선생님이 한 문제씩 ○×를 매기죠? 보상 모델은 로봇의 행동 영상을 보고 "성공! 90점!" 같이 자동으로 점수를 매깁니다. 사람이 일일이 보지 않아도 수천 개의 로봇 행동을 자동으로 평가할 수 있어요!

💡 왜 중요할까? — 로봇 학습에서 가장 어려운 부분 중 하나가 "이 행동이 성공인지 실패인지 판단하는 것"이었습니다. 보상 모델이 이 문제를 해결합니다.

5 🔄 개선하라 — DAgger 교정과 학습 인프라

로봇이 실패하면 사람이 직접 교정! 교정 데이터가 바로 다음 학습에 반영됩니다

👨‍🏫
개인 과외 선생님!

학생이 문제를 풀다가 막히면, 과외 선생님이 "여기서부터 이렇게 해봐" 하고 직접 보여주죠? DAgger(대거)도 같은 원리예요! 로봇이 작업하다 실패하면, 사람이 리더 팔로 "이렇게 해야 해" 하고 직접 교정해줍니다. 그 교정 동작이 자동으로 저장되어 바로 다음 학습에 사용됩니다!

💡 새 CLI: lerobot-rollout --strategy.type=dagger 한 줄로 이 모든 과정이 동작합니다. 배포 → 교정 수집 → 재학습의 "학습 바퀴(Flywheel)"가 CLI 플래그 하나로!

1 🦾 정책 배포 — 로봇이 학습된 정책으로 작업 수행
▼
2 ❌ 실패 감지 — 로봇이 실수! (사람이 버튼/페달 누름)
▼
3 👨‍🏫 사람 교정 — 리더 팔로 올바른 동작을 시연 교정 프레임에 자동으로 intervention 태그 부착
▼
4 💾 데이터 저장 — 교정 동작이 데이터셋에 자동 저장
▼
5 🔄 재학습 — 교정 데이터로 정책을 파인튜닝 배포 → 교정 → 학습 → 배포 → … 순환 반복!

Depth 지원이 왜 중요할까?

RGB 카메라만 있으면 로봇은 "사진"만 봐요. 물체가 가까이 있는지 멀리 있는지 정확히 모릅니다. Depth를 추가하면 "3D 입체 영화"처럼 거리 정보도 함께 알 수 있어요! Intel RealSense 카메라를 연결하고 use_depth: true 한 줄이면 끝!

💡 우리 랩과의 연결 — ZETA Satellite Robotics에서 PR #3644로 기여한 깊이맵 기능이 v0.6.0에 공식 포함되었습니다!

lerobot-annotate CLI를 사용하면, VLM(시각-언어 모델)이 로봇 에피소드 영상을 직접 보고 "지금 로봇이 빨간 큐브를 집고 있습니다"처럼 시간별 언어 설명을 자동으로 생성합니다. 이렇게 만들어진 풍부한 텍스트 주석은 미래의 언어 지시 기반 로봇 정책 학습에 핵심 재료가 됩니다.
